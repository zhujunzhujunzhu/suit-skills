## Web 控制台 · 翻译 CLI 配置（BUG-006 说明）

控制台「设置 → 翻译服务」选择 **本地 AI CLI** 时，后端会执行：

- `execFile(cliCommand, cliArgs, { timeout: 120000 })`，并把待译全文写入子进程 **stdin**，期望子进程在 **stdout** 输出译文（trim 后作为结果）。
- **无 shell**：Windows 下请使用已在 PATH 中的命令名，或填写可执行文件的**绝对路径**（如 `C:\\Tools\\my-translator.exe`）。
- 若你的 CLI 只支持「读文件」不支持 stdin，请自行写一个包装脚本：从 stdin 读入、写临时文件再调原 CLI，最后把 stdout 打印为译文。

**示例（示意，需与本机实际 CLI 一致）：**

| CLI 命令 | 附加参数 | 说明 |
|----------|----------|------|
| `npx` | `-y` `openai` `...` | 若该 CLI 支持从 stdin 读 prompt，可按其文档补全参数 |
| `python` | `C:\path\to\translate_stdin.py` | 自写脚本：`sys.stdin.read()` → 调用 API → `print(译文)` |

API 模式（OpenAI 兼容）一般比 CLI 更易配置，优先推荐。

## 📝 需求规格说明

### 端口占用自动重试

**问题描述：**
当前 `suit-skills web` 命令启动时，如果指定端口（默认 4587）被占用，会直接抛出错误：
```
EADDRINUSE: address already in use 127.0.0.1:4587
```

**需求目标：**
启动时检测端口占用，自动尝试 +1 递增的下一个端口，直到找到可用端口为止。

**技术细节：**

| 项目 | 说明 |
|------|------|
| 涉及文件 | `src/lib/web/server.ts`（`startWebServer` 函数）、`src/commands/web.ts`（CLI 命令） |
| 默认端口 | 4587 |
| 重试策略 | 端口 +1 递增 |
| 最大重试次数 | 3 次 |
| 用户指定端口 | `--port` 显式指定时也启用自动 +1 重试 |

**期望行为：**

1. 尝试在请求的端口启动服务
2. 如果收到 `EADDRINUSE` 错误：
   - 尝试 `port + 1`
   - 继续递增直到找到可用端口或达到最大重试次数
3. 如果超过最大重试次数仍失败，抛出错误
4. 启动成功后告知用户实际使用的端口

**用户提示示例：**
```
✓ Suit Skills Web started
Local: http://127.0.0.1:4589
(端口 4587 被占用，已自动尝试到 4589)
```

**边界情况：**
- 最大尝试 3 次（即最多试到 `requestedPort + 2`）
- 不超过端口范围上限 65535
