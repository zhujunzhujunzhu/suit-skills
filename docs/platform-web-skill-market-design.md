# ClawHub 技能市场设计文档

## 目标定位

ClawHub platform-web 的核心是一个中文技能市场。平台用于集中展示、检索、安装、评价和上传团队技能包。整体布局参考 local-web 的控制台体验：左侧导航承载模块切换，右侧主区域承载列表、详情和表单。

## 核心能力

### 技能市场

- 展示所有技能。
- 每个技能使用一行卡片，而不是网格卡片。
- 支持搜索、分类、来源、状态和排序。
- 点击技能行进入技能详情页。
- 技能行展示名称、描述、标签、作者、来源、版本、评分、评价数、安装量和更新时间。

### 技能详情

- 从技能市场点击进入详情。
- 支持返回技能市场。
- 展示技能说明、安装命令、版本、来源、作者、标签和状态。
- 展示评价统计和评价列表。
- 支持提交评价，评价绑定到当前技能。

### 评价能力

- 每个技能都可以评价。
- 评价字段包括评分、标签、文字反馈、匿名选项和联系方式。
- 管理端可以查看评价状态，并更新状态。
- 评价状态包括：新评价、处理中、已采纳、不采纳、已归档。

### 上传技能包

第一版先完成前端流程和状态，不要求立即接入真实后端上传。

- 支持上传 zip 包。
- 支持填写 Git 仓库地址。
- 支持后续从默认 Git 地址创建技能包。
- 表单字段包括技能名称、描述、分类、标签、作者、版本。
- 展示校验项：SKILL.md、metadata、命名冲突、敏感脚本、目录结构。
- 展示提交状态：待上传、校验中、校验通过、等待发布、已发布。

### 我的技能包

- 展示当前用户上传或维护的技能包。
- 展示发布状态、审核状态、Git 提交记录和更新时间。
- 后续可扩展编辑、撤回、重新发布。

### Git 配置

后续服务器上会使用 Git 登录账号和默认 Git 地址进行技能发布。

- 展示 Git 登录状态。
- 配置默认 Git 地址。
- 配置默认分支和技能包目录。
- 配置发布策略：直接提交、创建 Pull Request、管理员审核。
- 展示连接测试状态。

## 页面结构

```text
ClawHub
├─ 技能市场
├─ 上传技能
├─ 我的技能包
├─ 评价中心
└─ Git 配置
```

## 技能市场行卡片

每一行技能建议包含：

```text
[图标] 技能名称 / 描述 / 标签
作者 / 来源 / 版本 / 更新时间
评分 / 评价数 / 安装量 / 状态 / 查看详情
```

## 技能详情结构

```text
顶部：返回 / 技能名称 / 状态
主体：说明、安装命令、标签、元信息
评价：评分统计、评价列表、提交评价表单
```

## 后续接口建议

```text
GET    /api/skills
GET    /api/skills/:id
POST   /api/skills/upload
GET    /api/my-skills
POST   /api/evaluations
GET    /api/evaluations?skillId=
PATCH  /api/evaluations/:id/status
GET    /api/git/config
PUT    /api/git/config
POST   /api/git/test
```
