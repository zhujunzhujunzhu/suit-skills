# Windows 桌面端设置与源缓存需求

## 背景

Windows 桌面端出现“功能不可用”的表现，排查后发现主要集中在两类问题：

- CLI 的 warning / 进度提示混入 stdout，导致 `--json` 输出被污染，Tauri IPC 无法稳定解析。
- 技能库搜索和列表每次都会刷新远端源缓存，网络慢或源不可达时会让桌面端看起来卡住或不可用。

## 需求清单

1. 源缓存不应在每次搜索时刷新。
   - 默认每 5 分钟检查一次远端源。
   - 检查间隔内，搜索和列表优先使用本地缓存。
   - 手动点击刷新按钮时，必须立即强制检查远端源。
   - 检查间隔可以在设置中修改。
   - 间隔设置为 `0` 时，表示每次搜索都检查远端源。

2. 设置页应是独立路由页面，不应是弹框。
   - 顶部右上角设置按钮进入设置页。
   - 设置页通过 hash 路由访问，例如 `#settings`。
   - 设置页不出现在左侧主菜单中。
   - 旧的 `#agents` 链接需要兼容跳转到设置页。

3. Agents 目标位置管理应放入设置页。
   - 现有 Agent 目标路径管理能力保留。
   - 左侧主菜单只保留“技能库 / 已安装 / 源”等核心页面。
   - 技能详情中的“管理 Agents”入口应进入设置页。

4. 桌面端关闭行为应可配置。
   - 用户可以选择关闭按钮是退出窗口，还是隐藏到托盘。
   - 开启“关闭到托盘”后，点击关闭按钮隐藏窗口。
   - 点击托盘图标应恢复并聚焦主窗口。

5. 桌面端拖动区域需要更可靠。
   - 顶栏仍然可以拖动窗口。
   - 按钮、输入框、下拉框、窗口控制按钮等交互控件不能触发拖动。
   - 普通浏览器环境下应安全 no-op，不应报错。

6. 顶部刷新按钮需要反馈。
   - 点击刷新后，刷新按钮在请求期间禁用。
   - 刷新图标显示进行中动画。
   - 按钮暴露 `aria-busy` 状态，便于辅助技术感知。

7. JSON 输出必须保持可解析。
   - `--json` 命令的 stdout 只能输出机器可读 JSON。
   - warning、刷新提示、进度提示应进入 stderr。
   - Tauri IPC 解析层需要兼容旧输出中存在前置文本的情况。

## 当前实现记录

- 共享设置保存在 Suit Skills 配置的 `settings` 字段中。
- `settings.sourceRefreshIntervalMinutes` 控制源缓存检查间隔。
- `settings.minimizeToTray` 控制桌面端关闭按钮行为。
- Web API 通过 `/api/settings` 暴露设置读写能力。
- Tauri 桌面端通过 CLI config 命令读写同一份配置。
- Tauri 启动时创建托盘图标，左键点击托盘图标恢复主窗口。
- 设置页已迁移为 `#settings` 路由页面，并从左侧主菜单隐藏。

## 验收标准

- `npm run typecheck` 通过。
- `npm test` 通过。
- `cargo check` 通过。
- `npm run build:all` 通过。
- `npm run build:sidecar` 通过。
- `list --json` 的 stdout 以 JSON 开头，不包含 warning 或刷新提示。
- `#settings` 打开的是普通页面，而不是弹框或覆盖层。
- 顶部刷新按钮在刷新期间有禁用和旋转反馈。
