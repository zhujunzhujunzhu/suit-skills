# 端到端 / 代码审查发现的 Bug 清单

记录日期：2026-04-20  
范围：Skill 翻译、文件 API、详情页、设置中的翻译配置。

---

## 已修复

### BUG-001：顶栏面包屑在「技能详情」视图下显示错误

- **严重性**：🟡 警告  
- **现象**：`view === 'skill-detail'` 时落入最后的 `else`，显示为「设置」。  
- **修复**：在 `web/src/App.tsx` 面包屑链中增加 `skill-detail` 分支，文案键 `crumb.viewSkillDetail`（默认「技能详情」）。

### BUG-002：详情页仅在根目录查找 `SKILL.md`

- **严重性**：🟡 警告  
- **现象**：`data.files.find(...)` 只遍历顶层节点；若 `SKILL.md` 在子目录中会误报「未找到 SKILL.md」。  
- **修复**：新增 `findSkillMdInTree()`，递归遍历 `SkillFileNode` 树。

### BUG-003：翻译配置 PATCH 写入未规范化

- **严重性**：🟡 警告  
- **现象**：`updateWebTranslationConfig` 直接 `as TranslationConfig` 写入，可能留下与 `normalizeTranslationConfig` 不一致的脏字段。  
- **修复**：`src/lib/web/api.ts` 中合并后调用 `normalizeTranslationConfig` 再写入 `config.json`。

## 已知限制 / 未改（按优先级）

### BUG-004：设置里翻译表单项每次输入即 PATCH 配置（已修复）

- **严重性**：🔵 建议  
- **现象**：API Key 等字段每键入一字就请求 `PATCH /api/translation-config` 并频繁 toast。  
- **修复**：`SettingsView` 内使用 `translationDraft` 本地状态，仅点击「保存翻译设置」时调用 `onTranslationSave` 一次；文案与 i18n 已补齐。

### BUG-005：翻译接口请求体上限 1MB（已修复）

- **严重性**：🔵 建议  
- **现象**：`readJsonBody` 默认 1MB；超大 `SKILL.md` 可能 `413`。  
- **修复**：`readJsonBody(req, maxBytes)` 支持第二参数；`POST /api/translate` 使用 **8MB** 上限（`TRANSLATE_JSON_BODY_MAX_BYTES`）。

### BUG-006：CLI 翻译依赖本机可执行文件与环境（已文档化）

- **严重性**：🔵 建议  
- **现象**：`execFile(command, args)` 无 shell；需可执行文件在 PATH 或写绝对路径；进程需从 **stdin** 读入待译文本，将译文写到 **stdout**。  
- **修复**：在 [docs/other.md](other.md) 增加「Web 控制台 · 翻译 CLI 配置」说明与示例。

### BUG-007：详情页已简化为仅 `SKILL.md`（已恢复文件树）

- **严重性**：🔵 建议（产品）  
- **现象**：曾一度去掉左侧文件树。  
- **修复**：`SkillDetailView` 恢复 **左文件树 + 右内容区**；自动选中 `SKILL.md`（任意深度）并 **展开祖先目录**；无 `SKILL.md` 时仍可点选其它文件。
